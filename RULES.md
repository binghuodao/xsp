# XSP 交易规则文档

## 1. 核心参数

### 分数权重
| 指标 | 权重 | 分档（100 / 75 / 50 / 25 / 0） |
|------|------|----------------------------------|
| ADX  | 30%  | ≥30 / ≥25 / ≥20 / ≥15 / <15 |
| ER   | 20%  | ≥0.70 / ≥0.55 / ≥0.35 / ≥0.20 / <0.20 |
| BBW  | 15%  | ≥45 / ≥30 / ≥18 / ≥10 / <10 |
| Dev  | 15%  | ≥3.0 / ≥1.5 / ≥0.8 / ≥0.3 / <0.3 |
| VR   | 10%  | ≥2.0 / ≥1.3 / ≥0.8 / ≥0.5 / <0.5 |

- 公式：`score = 5 + Σ(分数(sub_val, T[k]) × W[k])`，四舍五入取整

### 关键阈值
| 参数 | 值 |
|------|----|
| is_trend | score ≥ 55 |
| 显示标签 Trending | score ≥ 65 |
| 近轨 ATR 乘数 | ATR14 × 0.60 |
| 近轨 BW 回退 | BW × 0.10（ATR14 不可用时） |
| 近轨方向分数下限 | score ≥ 50 |
| VIX 百分位阈值 | > 75 |
| 期权 DTE | 15 天 |
| 树 DTE | 7 天 |

## 2. 方向逻辑（4 层判断，从上到下）

### Level 1 — 趋势（非近轨 + is_trend）
```
非近轨 + score ≥ 55 → 趋势方向
  DI diff > 0  → CALL
  DI diff < 0  → PUT
  DI diff = 0  → 不开仓
```

### Level 2 — 近轨 + VIX 高 → 反转确认（覆盖趋势方向）
```
近轨 + score ≥ 50 + VIX% > 75
  贴 BB 上轨 → PUT
  贴 BB 下轨 → CALL
```

### Level 3 — 矛盾过滤
```
近 BB 上轨 + DI diff > 0 → 不开仓（趋势看多但价格在顶）
近 BB 下轨 + DI diff < 0 → 不开仓（趋势看空但价格在底）
```

### Level 4 — 近轨（原始）
```
近 BB 上轨 + score ≥ 50 → PUT
近 BB 下轨 + score ≥ 50 → CALL
近 BB 但 score < 50     → 不开仓
BB 中段                  → 不开仓
```

### 近轨判定
```
near_threshold = ATR14 × 0.60
near_top      = (BBU - price) < near_threshold
near_bottom   = (price - BBL) < near_threshold
```

| 方向 | 条件 | 结果 |
|------|------|------|
| PUT | SKEW < 140 | 不开仓（尾部风险低，不做空） |
| CALL | SKEW > 155 | 不开仓（尾部风险高，不做多） |

SKEW 来源：`^SKEW` via yfinance。

### 强度标签（仅用于显示）
| 标签 | 条件 |
|------|------|
| **strong** | score ≥ 72 + \|DI diff\| > 0 + skew_confirm |
| **normal** | score ≥ 65（非 strong） |
| **weak** | score < 65（有方向） |

### skew_confirm 判定
```python
skew_confirm = (direction == 'CALL' and SKEW < 145) or (direction == 'PUT' and SKEW > 145)
```

## 5. 树结构（Christmas Tree）

### 统一规则（7 DTE）
| 场景 | 中腿偏移 | S 腿 | L 腿 |
|------|----------|------|------|
| 趋势 + CALL | EMA20 + 5 | M - 10 | M + 5 |
| 趋势 + PUT | EMA20 - 5 | M + 10 | M - 5 |
| 震荡 | EMA20 ± 0 | M + 10 (PUT) / M - 10 (CALL) | M - 5 (PUT) / M + 5 (CALL) |

### 价格公式
```
PUT 树: 组合值 = S_mid + 2 × L_mid - 3 × M_mid
CALL 树: 组合值 = S_mid + 2 × L_mid - 3 × M_mid  （算法相同）
```

- 行权价均按 5 取整（`_s5()`）

## 6. 裸买（单腿）

- 仅当 `is_trend = True`（score ≥ 55）
- 7 DTE，Δ ≈ 0.35（CALL）或 Δ ≈ -0.35（PUT）
- 仅出现在 strong（最多 2 张）或 normal（1 张）等级

## 7. 持有计划

| 标的 | 持有期限 |
|------|----------|
| 全部（树/裸/ETF） | **t+5 收盘平仓** |

### 平仓触发条件（自动提示）
1. DTE ≤ 3
2. 盈利 ≥ 50% 或 亏损 ≥ 50%
3. BB 中段 + score < 65（建议减仓）
4. 趋势结束（上期 score ≥ 65 → 本期 < 65）
5. 方向冲突（持仓方向与当前信号不同）
6. 周末持仓风险（周五）
7. 浮亏达最大损失 80%
8. 方向切换（上期方向 ≠ 本期方向）

## 8. 风控规则（统一跟踪4%ETF + 固定止损兜底）

### 8.1 方案概览

| 层 | 触发条件 | 操作 | 说明 |
|----|---------|------|------|
| **统一跟踪** | peak × 0.96（从最高回落4%ETF） | 报告提示→手工平 | 主要风控，正常情况优先触发 |
| **固定止损（兜底）** | trend: entry×0.97 / nearBB: entry×0.99 | Moomoo自动成交 | 防开盘跳空，日常不会触及 |
| **t+5** | 第5天 | 手工平 | 到期平仓 |

有效止损价 = **min(固定止损价, 跟踪止损价)**。价格上涨时跟踪价随peak上移并低于固定价（正常跟踪先触发），下降时固定价兜底。

### 8.2 固定止损（Moomoo止蚀盘）
| 信号类型 | stop% | 触发价 | 覆盖风险 |
|---------|-------|--------|---------|
| **趋势** | **3.0%** | entry × 0.97 | 跳空暴跌 |
| **nearBB** | **1.0%** | entry × 0.99 | 反转失败 |

开仓一次性设定，无需后续修改。期权裸买/树不做止损。

### 8.3 统一跟踪（从最高ETF价回落4%）
```
跟踪价 = peak × 0.96
```
- `peak` = 持有期内最高 ETF 收盘价（每日更新，持久化至 position_tracker.json）
- 仅当 `peak > entry` 时跟踪价才有效（未盈利时只用固定止损）
- 不设自动单，每天看报告判断：现价 ≤ 跟踪价？手工平仓
- 上涨时跟踪价随peak上移并逐渐低于固定止损价（正常跟踪先触发）
- 3倍ETF: ETF 4% ≈ XSP 1.33%
- **整体 PF=1.79，年化 $3,386/yr（ETF），胜率 77.8%，最大回撤 2.2%**
- 历史 3 年跟踪触发率 60.1%，固定止损触发率 19.6%，到期平仓 20.3%

### 8.4 单笔最大亏损
最大仓位 $5,000（3x $15,000 名义）：

| 触发方式 | 最大亏损 | 说明 |
|---------|---------|------|
| 统一跟踪4% | $5k×4% = **$200** | 日常上限（从peak回落4%ETF≈XSP 1.33%） |
| 固定趋势3% | $5k×3% = **$150** | 跳空兜底 |
| 固定nearBB 1% | $5k×1% = **$50** | 跳空兜底 |

3年历史最差单笔 **$150**（即固定趋势止损）。本金额 $10,000。

### 8.5 执行流程
1. 开仓：买入 3x ETF，设 Moomoo 止蚀盘（trend: entry×0.97, nearBB: entry×0.99）
2. 不设止盈限价单
3. 每天看报告：有效止损价 = min(固定, 跟踪)；跟踪价触发→手工平；固定价触发→Moomoo自动平
4. 第 5 天手工平仓
