# XSP 交易规则文档

## 1. 核心参数

### 分数权重
| 指标 | 权重 | 分档（100 / 75 / 50 / 25 / 0） |
|------|------|----------------------------------|
| ADX  | 30%  | ≥30 / ≥25 / ≥20 / ≥15 / <15 |
| ER   | 20%  | ≥0.70 / ≥0.55 / ≥0.35 / ≥0.20 / <0.20 |
| BBW  | 15%  | ≥45 / ≥30 / ≥18 / ≥10 / <10 |
| Dev  | 15%  | ≥3.0 / ≥1.5 / ≥0.8 / ≥0.3 / <0.3 |
| VR   | 10%  | ≥2.0 / ≥1.3 / ≥0.8 / ≥0.5 / <0.5 |

- 公式：`score = 5 + Σ(分数(sub_val, T[k]) × W[k])`，四舍五入取整

### 关键阈值
| 参数 | 值 |
|------|----|
| is_trend | score ≥ 55 |
| 显示标签 Trending | score ≥ 65 |
| 近轨 ATR 乘数 | ATR14 × 0.60 |
| 近轨 BW 回退 | BW × 0.10（ATR14 不可用时） |
| 近轨方向分数下限 | score ≥ 50 |
| VIX 百分位阈值 | > 75 |
| 期权 DTE | 15 天 |
| 树 DTE | 7 天 |

## 2. 方向逻辑（4 层判断，从上到下）

### Level 1 — 趋势（非近轨 + is_trend）
```
非近轨 + score ≥ 55 → 趋势方向
  DI diff > 0  → CALL
  DI diff < 0  → PUT
  DI diff = 0  → 不开仓
```

### Level 2 — 近轨 + VIX 高 → 反转确认（覆盖趋势方向）
```
近轨 + score ≥ 50 + VIX% > 75
  贴 BB 上轨 → PUT
  贴 BB 下轨 → CALL
```

### Level 3 — 矛盾过滤
```
近 BB 上轨 + DI diff > 0 → 不开仓（趋势看多但价格在顶）
近 BB 下轨 + DI diff < 0 → 不开仓（趋势看空但价格在底）
```

### Level 4 — 近轨（原始）
```
近 BB 上轨 + score ≥ 50 → PUT
近 BB 下轨 + score ≥ 50 → CALL
近 BB 但 score < 50     → 不开仓
BB 中段                  → 不开仓
```

### 近轨判定
```
near_threshold = ATR14 × 0.60
near_top      = (BBU - price) < near_threshold
near_bottom   = (price - BBL) < near_threshold
```

### 强度标签（仅用于显示）
| 标签 | 条件 |
|------|------|
| **strong** | score ≥ 72 + \|DI diff\| > 0 + skew_confirm |
| **normal** | score ≥ 65（非 strong） |
| **weak** | score < 65（有方向） |

### skew_confirm 判定
```python
skew_confirm = (direction == 'CALL' and SKEW < 145) or (direction == 'PUT' and SKEW > 145)
```

## 5. 树结构（Christmas Tree）

### 统一规则（7 DTE）
| 场景 | 中腿偏移 | S 腿 | L 腿 |
|------|----------|------|------|
| 趋势 + CALL | EMA20 + 5 | M - 10 | M + 5 |
| 趋势 + PUT | EMA20 - 5 | M + 10 | M - 5 |
| 震荡 | EMA20 ± 0 | M + 10 (PUT) / M - 10 (CALL) | M - 5 (PUT) / M + 5 (CALL) |

### 价格公式
```
PUT 树: 组合值 = S_mid + 2 × L_mid - 3 × M_mid
CALL 树: 组合值 = S_mid + 2 × L_mid - 3 × M_mid  （算法相同）
```

- 行权价均按 5 取整（`_s5()`）

## 6. 裸买（单腿）

- 7 DTE，Δ ≈ 0.35（CALL）或 Δ ≈ -0.35（PUT）
- 仅出现在 strong（最多 2 张）或 normal（1 张）等级

## 7. 持有计划

| 标的 | 持有期限 |
|------|----------|
| 全部（树/裸/ETF） | **t+5 收盘平仓** |

### 平仓触发条件（自动提示）
1. DTE ≤ 3
2. 盈利 ≥ 50% 或 亏损 ≥ 50%
3. BB 中段 + score < 65（建议减仓）
4. 趋势结束（上期 score ≥ 65 → 本期 < 65）
5. 方向冲突（持仓方向与当前信号不同）
6. 周末持仓风险（周五）
7. 浮亏达最大损失 80%
8. 方向切换（上期方向 ≠ 本期方向）

## 8. 止损规则

### 按信号类型（宽止损防噪声，紧止损保反转）
| 信号类型 | stop% | 说明 |
|---------|-------|------|
| **趋势** | **3.0%** | 宽止损，仅拦截极端亏损（历史10.7%触发率） |
| **nearBB** | **1.0%** | 反转失败及时离场（历史51.6%触发率，PF=2.59） |

### 固定止损
```
ETF 止损价 = ETF 现价 × (1 - stop%)
```

- 直接填入 Moomoo 止蚀盘，无需换算
- 期权裸买/树不做止损，自行决定

### 追踪止损
当持仓盈利后自动触发，从最高价回吐一半利润时平仓：
```
trail_price = peak × (1 - stop% × 0.5)
```
- `peak` = 持有期内最高 ETF 收盘价（每日更新）
- 仅当 `peak > entry` 时激活
- 未达盈利前（peak ≤ entry）仅使用固定止损
- 追踪止损价 > 固定止损价时，用固定止损（更保守）
- 数据持久化至 `position_tracker.json`

### 单笔最大亏损
最大仓位 $5,000（3x $15,000 名义）。趋势：$5k×3×3% = **$450**；nearBB：$5k×3×1% = **$150**。历史3年最差单笔 **$270**。

本金额 $10,000。

### 执行流程
1. 开仓后立即挂 Moomoo 止蚀盘（止损价 ⬇️ 自动市价平）
2. 每天看报告更新止损价（追踪止损处于盈利时）
3. 持仓天数需检查（到期平仓）
